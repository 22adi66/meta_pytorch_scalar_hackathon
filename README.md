## 1. The Problem

There are **billions of lines of legacy C code** in production systems — operating systems, embedded firmware, network stacks — that are vulnerable to buffer overflows, null-pointer dereferences, and use-after-free bugs. Rust was designed to eliminate these problems at the language level.

**The catch:** Migrating a real C codebase to Rust is not a one-shot translation. It requires:

- Understanding **inter-file dependencies** (headers, shared types, function calls)
- Resolving **cascading compilation errors** across files
- Writing code that passes **functional tests** proving semantic equivalence
- Satisfying **architectural constraints** (no `unsafe`, low coupling)
- Doing all of this **in the right order** (you can't translate `data_store.c` before you've translated `math_ops.c` that it depends on)

No existing RL environment captures this complexity. CRust does.

---

## 2. The Environment

CRust implements the **OpenEnv interface** (reset / step / state / observation) as a live FastAPI server on Hugging Face Spaces with an NVIDIA A10G GPU.

### What the agent sees (observation)

```json
{
  "current_target": "math_ops.c",
  "c_source_code": "int add(int a, int b) { return a + b; } ...",
  "constraints": [
    "Do not use the unsafe keyword",
    "Maintain a CBO score below 3"
  ],
  "dependency_context": { "string_ops.rs": "pub fn concat(...)" },
  "recent_errors": [{ "level": "error", "message": "E0382: use of moved value" }],
  "phase": 1,
  "files_remaining": 3
}
```

### What the agent does (action)

```json
{
  "file_path": "src/math_ops.rs",
  "code_content": "pub fn add(a: i32, b: i32) -> i32 { a + b }"
}
```

### How reward is computed

The environment runs a **real Rust compiler** (`cargo check` + `cargo test`) inside a sandboxed Cargo workspace, then evaluates code quality metrics:

| Component | Weight | Signal |
|---|---|---|
| `cargo check` compiles | **0.30** | Structural correctness |
| `cargo test` passes | **0.30** | Semantic equivalence to C original |
| No `unsafe` blocks | **0.20** | Memory safety |
| CBO < 3 (low coupling) | **0.10** | Architectural quality |
| LCOM cohesion score | **0.10** | Module coherence |

**Anti-reward-hacking measures:** Protected test files (agent cannot modify ground-truth tests), subprocess timeouts (30s compile, 60s test), path traversal protection, `unsafe` keyword penalty (−0.50), CBO violation penalty (−0.20).

### Curriculum Learning (4 phases)

| Phase | Scope | Challenge |
|---|---|---|
| 1 | Single leaf-node file | Isolated function translation |
| 2 | Two-file dependency chain | Context-aware translation |
| 3 | Partial DAG with headers | Cross-file error resolution |
| 4 | Full repository | Long-horizon cascading error resolution |

The scheduler builds a **topological dependency graph** of the C codebase using Kahn's algorithm, ensuring files are migrated in dependency order (leaves first).

## 3. Training Results

We trained **Qwen/Qwen2.5-3B-Instruct** using **GRPO** (Group Relative Policy Optimization via TRL) directly on the A10G GPU of this HF Space.

![Reward Curve](reward_curve.png)
*Left: Reward over 100 GRPO training steps. Right: Per-component reward before vs after training. The agent goes from ~0.05 (random Rust) to **0.70** (compiles + memory-safe + good metrics) in ~25 steps.*

### Before training (zero-shot baseline)
- Generates Rust-looking code with syntax errors
- Often includes `unsafe` blocks
- Wraps code in extra `pub mod` nesting (wrong namespace)
- Average reward: **~0.05**

### After 100 steps of GRPO
```rust
// Generated by trained CRust agent — no unsafe, correct semantics
pub fn add(a: i32, b: i32) -> i32 { a + b }
pub fn subtract(a: i32, b: i32) -> i32 { a - b }
pub fn multiply(a: i32, b: i32) -> i32 { a * b }
pub fn divide(a: i32, b: i32) -> Option<i32> {
    if b == 0 { None } else { Some(a / b) }
}
pub fn clamp(value: i32, min_val: i32, max_val: i32) -> i32 {
    value.max(min_val).min(max_val)
}
```
- Compiles cleanly ✅
- No `unsafe` keyword ✅
- CBO = 0 (zero external dependencies) ✅
- Uses idiomatic Rust patterns (`Option<T>`, method chaining) ✅
- Average reward: **0.70** (+1,300% vs baseline)

**Trained LoRA adapter:** [`Adithyakommuri/crust-grpo-qwen25-3b`](https://huggingface.co/Adithyakommuri/crust-grpo-qwen25-3b) (114 MB)

## 4. Why This Matters

**Memory safety vulnerabilities cost the world billions of dollars annually.** The US NSA, CISA, and major tech companies have issued directives to migrate critical codebases to memory-safe languages like Rust. The bottleneck is human expertise — Rust is hard, especially for large codebases with complex dependencies.

An RL-trained LLM that can:
1. Understand C dependency graphs and migrate in topological order
2. Fix cascading compiler errors across multiple files
3. Produce code that passes semantic equivalence tests
4. Do this without human oversight

...could **automate years of migration work** for critical infrastructure. CRust is a proof-of-concept that this is trainable, not just prompting.

The environment is reusable: any C codebase can be dropped into the `legacy_c/` directory and the agent will build a dependency graph and migrate it systematically.

## Quick Start

### Test the live environment

```bash
# Get a migration task
curl -X POST https://adithyakommuri-meta-hackathon-final.hf.space/reset \
  -H "Content-Type: application/json" \
  -d '{"phase": 1, "constraints": ["Do not use the unsafe keyword"]}'

# Submit your Rust translation and get a reward
curl -X POST https://adithyakommuri-meta-hackathon-final.hf.space/step \
  -H "Content-Type: application/json" \
  -d '{"file_path": "src/math_ops.rs", "code_content": "pub fn add(a: i32, b: i32) -> i32 { a + b }"}'
```

**Swagger UI:** https://adithyakommuri-meta-hackathon-final.hf.space/docs

### Reproduce training (Colab T4, free)

Open [`CRust_Training_Colab.ipynb`](CRust_Training_Colab.ipynb) in Google Colab and run all cells.

### Start training on the Space GPU (A10G)

```bash
# Launch GRPO training on the Space's A10G directly
curl -X POST https://adithyakommuri-meta-hackathon-final.hf.space/train/start \
  -H "Content-Type: application/json" \
  -d '{"max_steps": 100, "model_name": "Qwen/Qwen2.5-3B-Instruct", "phase": 1}'

# Monitor live progress
curl https://adithyakommuri-meta-hackathon-final.hf.space/train/status
```

### Load the trained model

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel
import torch

model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-3B-Instruct", dtype=torch.bfloat16, device_map="auto"
)
model = PeftModel.from_pretrained(model, "Adithyakommuri/crust-grpo-qwen25-3b")
```
